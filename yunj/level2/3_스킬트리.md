# 3_스킬트리
```
import java.util.Stack;

class Solution {
    public int solution(String skill, String[] skill_trees) {
        int answer = 0;
        Stack<Integer> stack = new Stack<>();
        char[] skillArr = skill.toCharArray();
        int tmp = -1;
        boolean no = true;
        for(String user_sk : skill_trees){
            for(char sa : skillArr){
                stack.push(user_sk.indexOf(sa));
            }

            while (!stack.isEmpty()){
                if(tmp == stack.peek()){
                    stack.pop();
                } else if(tmp == -1){
                    tmp = stack.pop();
                } else if(tmp > stack.peek() && stack.peek() != -1){
                    tmp = stack.pop();
                } else{
                    no = false;
                    break;
                }
            }
            if (no){
                answer++;
            }
            tmp = -1;
            no = true;
            stack.clear();
        }
        return answer;
    }
}
```

유저의 스킬트리인 skill_trees 를 순환한다.

유저 스킬트리에서 스킬트리들의 인덱스를 찾는다. 

만약 인덱스가 오름차순이라면 허용되는 스킬트리이다. 

선행스킬만 배웠다면 후행스킬은 배우지 않아도 무방하다.

---

> 선행스킬만 배웠다면 후행스킬은 배우지 않아도 무방하다.

인덱스 stack에 -1 이 섞여있을 수 있다.   pop하였을 때 최초의 연속적인 -1 은 후행스킬을 배우지 않으것임으로 괜찮지만 중간에 섞여 있는 -1 을 스킬트리 오류이다.
